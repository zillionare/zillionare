## 前言：为什么你应该关心因果卷积？

想象一个场景：

> 2024年某天，你看到一只股票连续上涨了5天。你会不会觉得"明天还会涨"？

大多数人的直觉是**会**。但如果你把自己放在量化研究员的位置，会更在意另外几件事：这个“连续上涨”在统计上到底代表什么，我们能不能用可计算的数学表达把它写出来，以及如果要让模型自己学习“什么时候跟随趋势、什么时候规避回撤”，该把什么样的结构喂给它。

今天要介绍的**因果卷积**，就是处理这类问题的一把实用工具。你可以把它理解成：把过去一段价格、成交量、波动这些信息，按重要性重新加权，然后压成一个能用于决策的信号。它听起来像深度学习术语，但落到交易里，其实非常接地气。

这篇文章只做三件事：先把因果卷积用交易语言讲清楚，再把它和 MA、EMA、MACD 这些你熟悉的东西接起来，最后再聊为什么 TCN 在很多序列任务里，确实比一些老方法更稳、更快。

## 先用一个交易直觉把它讲明白

很多人刚入市时都经历过同一种路径：先靠感觉交易，看到连涨就追，看到连跌就躲，结果常常是追在高位、卖在低位。后来开始学指标，知道看 MACD、KDJ，也能意识到“涨了”不是最关键，真正关键是涨速有没有变、动量有没有衰减。

再往前一步，你会发现光盯一个指标还是不够，因为市场同时在发生短期波动、中期趋势和长期均值回归。真正稳一点的做法，是把这些信息放进一个统一框架，让模型自己学每种信号该给多大权重。**因果卷积**就是干这件事的核心工具：它不靠拍脑袋判断，而是把“经验”变成可计算、可训练的结构。

你可以先这样记：

> **因果卷积，就是用"加权求和"的方式，把过去一段时间的数据，整合成一个有意义的数值。**

其实就是先回看一段历史窗口，比如最近 5 天或 20 天；然后给这段历史里的每一天分配不同权重，通常离今天更近的数据会更重要；最后把这些信息压成一个分数，用来辅助你判断当前该偏多、偏空，还是观望。

换个更生活化的类比：

想象一个基金经理的年终评分：
最终得分 = 0.30×年化收益率 + 0.25×夏普比率 + 0.20×最大回撤 + 0.15×规模 + 0.10×流动性
= 0.30×15% + 0.25×1.8 + 0.20×(-8%) + 0.15×50亿 + 0.10×80%
      = 4.5 + 0.45 - 1.6 + 7.5 + 8%
      = 10.85分（满分大概15分左右）
这个公式本质上就是一个**卷积**：

输入是收益率、夏普比率、回撤、规模、流动性这些指标；权重是 `[0.30, 0.25, 0.20, 0.15, 0.10]`；最后输出一个综合得分。因果卷积做的事情和它完全同构，只不过它处理的是按时间排列的数据序列。

**因果卷积就是：把这个思想应用到"时间序列"上。**

## 从你熟悉的指标出发

你在交易软件里最常见的 MA5、MA10、MA20，本质上都是同一件事：把一段历史价格做加权求和。拿 MA5 举个最简单的例子，假设最近 7 天价格是 `100, 102, 101, 105, 107, 106, 108`，那 Day5 的 MA5 就是前 5 天平均后得到 `103.0`，Day6 会滚动成 `104.2`，Day7 变成 `105.4`。看起来像“平均”，其实已经是卷积思路了，只不过这时候权重是均等的 `[1/5, 1/5, 1/5, 1/5, 1/5]`。

EMA 比 MA 更贴近真实交易直觉，因为它默认“越近的数据越值得相信”。它的写法是 `EMA_t = α × Price_t + (1-α) × EMA_{t-1}`，展开后就是一串指数衰减的加权求和：今天权重最大，昨天次之，越往前权重越小。

比如取 `α=0.3`，当前价格权重是 `0.3`，前一天是 `0.21`，再往前是 `0.147`、`0.103`、`0.072`……你会看到一个很清晰的结构：旧信息不会被直接丢掉，但影响力会按指数速度递减。这正是很多趋势策略偏爱 EMA 的原因，也是因果卷积在时间维度上做加权的直觉基础。

MACD 你肯定见过，但把它换个视角看会更有意思。我们平时写成 `MACD Line = EMA(12) - EMA(26)`，再对它做一次 `EMA(9)` 得到信号线，最后用两者的差形成柱状图。看起来像是“指标公式套指标公式”，其实底层逻辑非常统一：先做两次不同窗口的加权求和，再看它们的差值。换句话说，MACD 本质上就是“两个卷积结果的差”。

## 再把直觉落到数学上

对于一个长度为 $T$ 的时间序列 $\{x_1, x_2, \ldots, x_T\}$，**因果卷积**定义为：

$$
h_t = \sum_{k=0}^{K-1} f[k] \cdot x_{t-k}
$$

这里的 $h_t$ 是时刻 $t$ 的输出，$f[k]$ 是卷积核在位置 $k$ 的可学习权重，$K$ 对应回顾窗口长度。

这里最关键的是“因果性”：

$$
h_t \text{ 只依赖于 } \{x_1, x_2, \ldots, x_t\}
$$

即：**输出只用当前和过去的信息，不使用未来数据。**


如果你对公式不敏感，看这张图就够了：

```
时间:      t=1     t=2     t=3     t=4     t=5     t=6
│       │       │       │       │       │
输入 x:   [x₁]    [x₂]    [x₃]    [x₄]    [x₅]    [x₆]
│       │       │       │       │       │
│   ┌───┴───┐   │       │       │       │
卷积核 k=3:   │ [f₀,f₁,f₂] │       │       │       │
│   └─┬─┬─┬─┘   │       │       │       │
│     ↓↓↓       │       │       │       │
输出 h:           h₃       h₄      h₅      h₆
←──────────────→
 只用过去和现在

h₃ = f₀·x₃ + f₁·x₂ + f₂·x₁
h₄ = f₀·x₄ + f₁·x₃ + f₂·x₂
h₅ = f₀·x₅ + f₁·x₄ + f₂·x₃
h₆ = f₀·x₆ + f₁·x₅ + f₂·x₄
```

只用 $K=3$ 的卷积核时，模型每次只能看很短一段历史，所以会引入**膨胀（dilation）**来把视野拉长。它的定义是：

$$
h_t = \sum_{k=0}^{K-1} f[k] \cdot x_{t-d \cdot k}
$$

其中 $d$ 是膨胀率。

dilation=2 时大概是这个效果：

```
普通卷积 (d=1):  ★──★──★──★──★──★──★──★    感受野=3
                 ↑
膨胀卷积 (d=2):  ★─────★─────★─────★─────★    感受野=5
                 ↑
d=4:             ★─────────★─────────★        感受野=9
```

多层叠起来后，感受野会快速变大：

```
Layer 1 (d=1):  ★──★──★──★──★──★──★──★    感受野=3
Layer 2 (d=2):    ★──★──★──★──★──★──★──★  感受野=7
Layer 3 (d=4):      ★──★──★──★──★──★──★──★感受野=15
Layer 4 (d=8):        ★──★──★──★──★──★──★──★感受野=31
```

写成公式就是：

$$
R_L = 1 + (K-1) \cdot (2^L - 1)
$$

其中 $L$ 是层数。

## 放到金融建模里看会更清楚

如果你熟悉 ARIMA 那一套，可以把它和因果卷积这样理解：传统时间序列更像先做一堆假设，再去拟合参数；因果卷积则是让模型直接从数据里学权重。前者在线性关系上很清晰，但遇到非线性和长依赖时往往要做很多手工处理；后者用膨胀卷积就能覆盖更长历史，而且天然支持并行，序列一长优势会更明显。

设 $\{X_t\}$ 是一个随机过程（股价序列），因果卷积的输出 $\{H_t\}$ 定义为：

$$
H_t = \sum_{k=0}^{\infty} f[k] \cdot X_{t-k}
$$

前提条件是 $\{f[k]\}$ 绝对可和，即 $\sum_{k=0}^{\infty} |f[k]| < \infty$。

直白一点说，这说明输出是输入历史信息的一种线性组合。放到信号处理语境里，它就和线性滤波器是同一类对象，也就是常说的线性时不变系统（LTI）。

你也可以把它和卡尔曼滤波放在一块看。卡尔曼的状态更新一般写成：

$$
\hat{X}_t = A\hat{X}_{t-1} + B y_t
$$

而因果卷积写成：

$$
H_t = \sum_{k} f[k] \cdot X_{t-k}
$$

卡尔曼滤波和因果卷积本质上都在做“线性组合过去信息”。区别在于前者是递归更新，权重来自状态空间模型；后者是显式加权和，权重由神经网络从数据里学出来。

放到波动率建模里，对比会更直观。GARCH(1,1) 通常写成：

$$
\sigma_t^2 = \omega + \alpha \epsilon_{t-1}^2 + \beta \sigma_{t-1}^2
$$

TCN 的一类写法是：

$$
\sigma_t = \sum_{k=0}^{K-1} f[k] \cdot r_{t-k}^2
$$

一句话对比：GARCH 是**固定结构**的加权（递归），TCN 是**可学习结构**的加权（灵活）。

## 为什么“加权求和”这条路走得通

先借用一个听起来有点学院派的词：万能逼近定理。它想表达的其实很朴素：只要网络容量足够，"加权求和 + 非线性激活"这套组合就能逼近很复杂的函数关系。放到交易里就是，哪怕涨跌背后的关系很绕，模型也不一定学不会，这也是为什么卷积层、全连接层看起来都在做加权求和，却依然能在复杂任务里跑出效果。

如果你希望有一个严格表达，它对应的数学形式是：

$$
\forall \epsilon > 0, \exists N, f_i, w_i, b_i: \left| g(x) - \sum_{i=1}^{N} w_i \cdot \sigma(f_i \cdot x + b_i) \right| < \epsilon
$$

这件事也能和金融里熟悉的线性直觉接上。我们平时本来就在用 CAPM、APT、线性因子模型这些框架，因果卷积并不是把它们推翻，而是在这层直觉上再往前走一步：权重可以学习，表达可以非线性，所以拟合复杂关系的能力会更强。

最后回到实盘研究里最现实的问题——算力。RNN 往往按时间步串行计算，复杂度常写成 $O(T \cdot D^2)$；TCN 用并行卷积，常见写法是 $O(T \cdot K \cdot C \cdot \log T)$。序列一长，训练和回测速度差距会很明显，很多时候这也是 TCN 在工程上更顺手的原因；说直白点，在长序列里它确实经常会比 RNN 快出一个量级。

## 代码里它到底怎么写

```python
import torch
import torch.nn as nn

class CausalConv1D(nn.Module):
    """
    因果卷积层
    确保输出只依赖于当前及之前的数据
    """
    def __init__(self, in_channels, out_channels, kernel_size, dilation=1):
        super().__init__()
        self.kernel_size = kernel_size
        self.dilation = dilation
        
        # 关键：padding = (kernel_size - 1) × dilation
        # 这样可以保持序列长度，同时确保因果性
        self.padding = (kernel_size - 1) * dilation
        
        self.conv = nn.Conv1d(
            in_channels, 
            out_channels, 
            kernel_size,
            padding=self.padding,  # 左侧填充
            dilation=dilation     # 膨胀率
        )
    
    def forward(self, x):
        """
        x: (batch, channels, length)
        """
        x = self.conv(x)
        
        # 裁剪右侧：移除因padding产生的多余输出
        if self.padding > 0:
            x = x[:, :, :-self.padding]
        
        return x

# 测试
batch_size = 32
in_ch = 1
out_ch = 16
seq_len = 100

x = torch.randn(batch_size, in_ch, seq_len)
conv = CausalConv1D(in_ch, out_ch, kernel_size=3, dilation=1)
y = conv(x)

print(f"输入形状: {x.shape}")      # [32, 1, 100]
print(f"输出形状: {y.shape}")     # [32, 16, 100]
print(f"因果性验证: ✓ 输出只依赖输入的前3个时间点")
```

class ResidualBlock(nn.Module):
    """
    TCN 残差块：包含两个因果卷积 + 残差连接
    """
    def __init__(self, channels, kernel_size=3, dilation=1, dropout=0.2):
        super().__init__()
        
        # 第一个因果卷积
        self.conv1 = CausalConv1D(channels, channels, kernel_size, dilation)
        self.ln1 = nn.LayerNorm(channels)
        
        # 第二个因果卷积
        self.conv2 = CausalConv1D(channels, channels, kernel_size, dilation)
        self.ln2 = nn.LayerNorm(channels)
        
        self.dropout = nn.Dropout(dropout)
        
        # 投影层（维度不匹配时使用）
        self.shortcut = nn.Identity()
    
    def forward(self, x):
        """
        x: (batch, channels, length)
        """
        residual = self.shortcut(x)
        
        # 第一个卷积块
        out = self.conv1(x)

```python
class StockPredictorTCN(nn.Module):
    """
    基于 TCN 的股价预测模型
    输入：过去5天的 [开盘价, 最高价, 最低价, 收盘价, 成交量]
    输出：明天的涨跌概率
    """
    def __init__(self, input_dim=5, hidden_dim=64, num_classes=2):
        super().__init__()
        
        # 初始投影层
        self.input_conv = nn.Conv1d(input_dim, hidden_dim, 1)
        
        # TCN 残差块（膨胀率：1, 2, 4, 8）
        self.tcn_blocks = nn.ModuleList([
            ResidualBlock(hidden_dim, kernel_size=3, dilation=1),
            ResidualBlock(hidden_dim, kernel_size=3, dilation=2),
            ResidualBlock(hidden_dim, kernel_size=3, dilation=4),
            ResidualBlock(hidden_dim, kernel_size=3, dilation=8),
        ])
        
        # 输出层
        self.output_conv = nn.Conv1d(hidden_dim, hidden_dim // 2, 1)
        self.classifier = nn.Linear(hidden_dim // 2, num_classes)
    
    def forward(self, x):
        """
        x: (batch_size, seq_len, features)
        输出: (batch_size, num_classes) 涨跌概率
        """
        # (B, L, F) -> (B, F, L)
        x = x.transpose(1, 2)
        
        # 初始投影
        x = self.input_conv(x)
        
        # TCN 残差块
        for block in self.tcn_blocks:
            x = block(x)
        
        # 取最后一个时间点
        x = x[:, :, -1]  # 此时形状变为 (B, C)，即 2D 张量
        
        # 修正点：x 已经是 2D 张量，不能再使用 transpose，直接传入分类器
        # 先通过输出层调整通道数，这里如果是 Linear 层需要注意维度
        # 因为 x 已经是 (B, C)，所以直接用 Linear 替代 output_conv 会更合理，
        # 但为了保留原有结构，可以将其恢复为 3D 再卷，或者直接展平
        x = x.unsqueeze(-1)  # 变回 (B, C, 1) 以便使用 Conv1d
        x = self.output_conv(x).squeeze(-1) # 变回 (B, C')
        return self.classifier(x)


# 使用示例
model = StockPredictorTCN(input_dim=5, hidden_dim=64, num_classes=2)

# 模拟输入：batch=32, 序列长度=5, 特征=5
x = torch.randn(32, 5, 5)  # [开盘, 最高, 最低, 收盘, 成交量] × 5天
output = model(x)  # [32, 2]  每行是 [P(跌), P(涨)]

print(f"输入: {x.shape}")
print(f"输出: {output.shape}")
print(f"预测结果: {torch.softmax(output, dim=1)[:3]}")
```

## 和常见方法放一起比一比

横向看几个常见方法：简单移动平均上手快，但表达能力有限，更多是做平滑；ARIMA 在线性序列上依然能打，不过对复杂非线性场景不够灵活；LSTM 能学非线性和依赖关系，但训练速度和并行性是硬伤；TCN 兼顾了非线性建模、长依赖捕捉和并行计算，在很多时间序列任务里更容易做到“效果和效率都不太差”。

一句话说，TCN 把很多人最在意的两件事放在了一起：一边是并行计算，训练和推理都更利索；另一边是通过膨胀卷积把依赖范围拉长，不会只盯着最近几根 K 线。相比之下，RNN 往往要按时间步串行推进，序列长了以后效率和梯度稳定性都更容易出问题。再加上 TCN 的感受野可以通过层数和膨胀率直接调，做策略时你能更明确地控制“模型到底在看多长的历史”。

---

## 往前看：这条线还在怎么发展

这条路线的发展其实很连续：2016 年 WaveNet 把膨胀因果卷积做出了代表性结果；2018 年 Bai 等人的 TCN 论文把“卷积能不能替代循环网络”这件事讨论得很完整；再往后，不少工作开始把 TCN 和注意力机制结合。到现在，TCN 不是唯一答案，但它已经是时间序列建模里绕不开的一条主线。真正还值得继续啃的，是上限问题、超参数自动化问题，以及可解释性问题：卷积核到底在市场里学到了什么模式，怎么把它讲成人能听懂的话。

---

## 结语：回到最初的问题

文章开头问：

> **一只股票连续上涨5天，明天还会涨吗？**

如果用因果卷积来回答，这个问题就不再是“猜”，而是“算”。模型会把过去几天的信息压成一个模式分数，里面既有动量，也有波动，还能随着数据变化自己调整权重。

TCN 进一步做的，是把这种判断放到多个时间尺度上：短层看近几天节奏，深层看更长的结构。最后给你的不是一句“涨或跌”的口号，而是一个概率，比如 $P(\text{明天上涨}\mid\text{历史模式})$，然后你再把它接到仓位、止损和风控上。

说到底，因果卷积最有用的地方，不是神奇预测，而是把“经验交易”变成“可复盘、可迭代、可验证”的流程。这一步做对了，策略才有机会从回测走向实盘。
