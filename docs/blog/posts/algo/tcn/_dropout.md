# Dropout：AI模型的"断臂求生"之道

你有没有遇到过一个让人崩溃的情况：训练集准确率98%，测试集92%，看着挺好，结果一上实盘直接亏30%？

2022年就有量化基金踩了这个坑。回测数据漂亮得不像话，模拟盘年化45%，结果第一年实盘亏了30个点。基金经理满脸问号：明明测试集表现很好，为什么实盘亏成狗？

答案很简单：**过拟合了**。

---

## 过拟合是什么

说白了就是死记硬背，考试换个题型就不会。

AI模型干了什么事？它把2015年到2021年的数据都背下来了：茅台出财报当天都涨，降准后银行股都涨，宁德时代销量公布当天必涨。结果2022年全反了：这些老经验统统失效，AI完全懵逼。

这就像你背熟了所有历史真题，但高考一换新题型就完蛋。

---

## Dropout是什么

Dropout要做的事很简单：训练的时候，随机关掉一些神经元。

正常训练是所有神经元一起干活，但问题在于某些神经元会变懒——反正有别人在，我不用学那么深也能混过去。一旦某个关键神经元出问题，整个网络就废掉了。

Dropout的思路是：每次训练随机抽掉一部分人，剩下的人必须学会配合，不然打不赢。类比一下就像篮球队训练，不能总靠5个主力打天下，得让替补也上场练练，不然主力一受伤整个队就完了。

换到神经网络上也是这个道理：每个神经元都得学点备用技能，不能只依赖某一个。整个网络因此变得更抗揍。

---

## 具体怎么工作

没有Dropout的时候，网络里所有神经元一起参与计算。如果某个神经元特别强，其他人就容易变懒。一旦这个强者出了问题，整个网络就瘫痪了。

加上Dropout之后，训练变成了这样：

第一轮训练，随机关掉A神经元，只剩B和C在工作，它们被迫多干活；下一轮训练，关掉B神经元，A和C就得承担更多；再下一轮关掉C，那么A和B就得学会补位。这样每个神经元都得练出备用技能，整个网络不会因为某个节点失效而崩溃。

到了测试或者实盘的时候，Dropout会关闭，所有神经元都参与进来，把之前学的备用技能都用上。

---

## 关键参数怎么选

Dropout有个核心参数叫keep_prob，意思是保留多少神经元。0.5意味着一半神经元在工作，一半被关掉。数值越大越接近正常训练，太大了防不住过拟合；太小了网络太残，什么都学不会。

金融数据噪声大，容易过拟合，通常推荐0.3到0.5之间。股票选股可以狠一点设到0.3到0.5，股价预测设0.2到0.4，高频策略因为数据规律相对稳定可以设低一点0.1到0.2。如果是样本很少的小数据集，就得多关掉一些，0.4到0.6会比较稳妥。

有一点必须记住：Dropout只在训练时用。测试和实盘的时候一定要调用model.eval()，让Dropout关闭，让模型发挥全部能力。很多人实盘翻车就是这个原因——忘了切换到评估模式，模型还在随机关神经元。

---

## Dropout能解决什么

它解决的是过拟合问题。强制每个神经元都学点备用技能，不把鸡蛋放在一个篮子里。

但它不是万能的。数据质量本身有问题，Dropout救不了；模型结构设计得不合理，Dropout也救不了；如果模型本来就太简单、学什么都不够（欠拟合），Dropout只会让情况更糟。

判断要不要用Dropout有个简单方法：训练集准确率很高但测试集很低，说明过拟合了，需要Dropout；训练集和测试集准确率都很低，那是欠拟合，用Dropout只会更糟。

---

## 一些实战技巧

Dropout值从0.1开始试，慢慢往上加，找到效果最好的那个就行。也可以考虑分层设置，Embedding层设低一点0.1，中间层适中0.3，输出层可以高一点0.5。

数据量的影响也很大：数据多的时候不容易过拟合，Dropout设0.1到0.2就够了；数据少的时候容易过拟合，Dropout可以设到0.4到0.6。

## 代码里怎么写

PyTorch里用Dropout特别简单：

```python
import torch.nn as nn

model = nn.Sequential(
    nn.Linear(64, 128),
    nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(128, 32),
    nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(32, 2)
)

model.train()  # 训练时打开Dropout
model.eval()   # 测试/实盘时关闭Dropout
```

如果用Module写法：

```python
class StockModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.lstm = nn.LSTM(5, 64, num_layers=2, batch_first=True)
        self.dropout = nn.Dropout(0.3)
        self.fc = nn.Linear(64, 1)

    def forward(self, x):
        lstm_out, _ = self.lstm(x)
        out = lstm_out[:, -1, :]
        out = self.dropout(out)
        out = self.fc(out)
        return out
```

## 还有哪些变种

Spatial Dropout和普通Dropout的区别在于：普通Dropout关掉单个神经元，Spatial Dropout关掉一整个通道。比如股价数据里，开盘、最高、最低、收盘是一个通道，Spatial Dropout会把这4个一起关掉，在某些场景下效果更好。

DropConnect则是关掉神经元之间的连接而不是关掉神经元本身。士兵还在但通讯断了，和直接让士兵请假不太一样。不过这个用得比较少。

---

## 一句话理解Dropout

Dropout就是"断臂训练"——活着比什么都重要。

没有Dropout的时候，AI死记硬背，测试集厉害但实盘亏钱。有Dropout的时候，AI被逼着随机缺胳膊少腿地训练，每个神经元都得学备用技能。换题型也不怕。

所以金融AI模型里Dropout几乎是标配，因为金融数据噪声大、规律复杂，太容易过拟合了。设个0.3到0.5，花点时间调一调参数，比实盘亏30个点划算多了。
