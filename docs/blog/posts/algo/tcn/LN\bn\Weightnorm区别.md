# BN、LN、WN 三个归一化方法到底有什么区别？

> 一张图 + 三个公式 + 三个场景，彻底讲清楚

---

## 先看一张图

```
BN（Batch Norm）:  ← 横向比：把所有人的同一科成绩放一起算均值
    语文：老王80分，全班平均75分 → 老王偏高
    数学：老王60分，全班平均85分 → 老王偏低

LN（Layer Norm）:  ← 纵向比：把自己各科成绩放一起算均值
    老王：语文80，数学60，英语70 → 自己的平均是70分

WN（Weight Norm）:  ← 不比成绩，而是调整教练的教学方式
    不管学生考多少分，直接调整教练怎么教
```

---

## 一句话概括

| 方法 | 归一化的是什么 | 跟谁比 |
|------|--------------|--------|
| **BN** | 同一层的输入数据 | 跟batch里其他人比 |
| **LN** | 同一层的输入数据 | 跟自己比 |
| **WN** | 模型权重参数 | 不比数据 |

---

## BN、LN：都是"归一化数据"，区别在于"跟谁比"

假设我们有一个batch的数据：

```
样本1（茅台）：[1800, 1850, 1900]   → 三天的收盘价
样本2（工行）：[5.0, 5.1, 5.3]      → 三天的收盘价
样本3（腾讯）：[400, 410, 420]      → 三天的收盘价
```

### BN怎么算？

**对"第一天"这个时间点，把三只股票的价格混在一起算均值和方差：**

```
均值 = (1800 + 5 + 400) / 3 = 735
方差 = 三只股票第一天价格的离散程度
```

然后归一化：`x' = (x - 均值) / 标准差`

**问题**：1800和5放一起算均值，完全没意义！

### LN怎么算？

**对"茅台"这只股票，把它的三天价格自己算均值和方差：**

```
均值 = (1800 + 1850 + 1900) / 3 = 1850
方差 = 茅台自己三天价格的离散程度
```

**优点**：每只股票只跟自己比，不受其他股票价格影响。

---

## WN：跟BN/LN根本不是一回事

WN（Weight Normalization）的核心思想是：

> **不归一化数据，而是归一化模型权重**

### 原来的全连接层

```
y = Wx + b
```

W是一个矩阵，比如形状是 `[512, 784]`

### WN的做法

把 W 拆成两部分：

```
W = g * (v / ||v||)

其中：
  v 是方向向量，可学习
  g 是尺度标量，可学习
  ||v|| 是 v 的模（长度）
```

**本质**：W的"方向"由v决定，W的"长度"由g决定。这样做的好处：

1. **梯度更稳定**：||v||的大小被归一化了，梯度不会爆炸
2. **不依赖batch**：完全没有用到数据的统计量
3. **训练更快收敛**：权重更新更有效率

## 三个场景说明白

### 场景1：同时选茅台、工商银行、腾讯

| 方法 | 会发生什么 |
|------|-----------|
| **BN** | 茅台1800，工行5块，腾讯400 → 平均735 → 工行被判定为"极端低价" → **错误！** |
| **LN** | 每只股票只跟自己比 → 看涨跌幅 → **正确！** |
| **WN** | 不管价格多少，只管模型权重调没调好 → **训练更稳定** |

### 场景2：训练数据很少，batch只有2个样本

| 方法 | 能用吗 |
|------|--------|
| **BN** | ❌ 不能，2个样本算均值方差没意义 |
| **LN** | ✅ 可以，单个样本内部归一化，不依赖batch大小 |
| **WN** | ✅ 可以，根本不依赖数据统计量 |

### 场景3：模型训练震荡不收敛

| 方法 | 有帮助吗 |
|------|---------|
| **BN** | 可能有帮助，但不解决根本问题 |
| **LN** | 可能有帮助 |
| **WN** | ✅ 最有帮助，直接让权重更新更稳定 |

---

## 一个公式总结

```
BN: y = (x - E[x_batch]) / sqrt(Var[x_batch]) * γ + β
    → x是batch里所有人的同一特征

LN: y = (x - E[x_sample]) / sqrt(Var[x_sample]) * γ + β
    → x是我自己的所有特征

WN: y = (g * v / ||v||) · x + b
    → W被重新参数化了，跟x没关系
```

---

## 到底用哪个？

**用BN：**
- batch足够大（≥32）
- 同一行业内部比较（比如都是银行股）
- CNN图像任务（BN是图像领域的默认选择）

**用LN：**
- 时间序列预测（股票、语音等）
- 跨行业比较（茅台vs工商银行）
- 小batch或online learning
- Transformer/Attention架构

**用WN：**
- 训练不稳定、梯度爆炸
- 小数据场景
- 想要结合BN/LN使用（WN不改变数据分布，可以和LN叠加）

**TCN/量化场景：**
- 主力用LN（时间序列 + 跨行业选股）
- 配合WN让训练更稳定
